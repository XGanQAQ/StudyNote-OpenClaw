单例模式使用 “三做四不做” 速记表  
（背完即可在 Unity/大型项目里 90 % 不踩坑）

------------------------------------------------  
【三做】  
1. **只放只读数据**  
   ‑ 配置表、常量、版本号、数学公式；字段全部 `readonly`。  

2. **必须私有构造 + 静态只读属性**  
   ```csharp
   public sealed class AudioConfig
   {
       public static AudioConfig I { get; } = new AudioConfig();
       private AudioConfig() { }
   }
   ```

3. **提供 Reset / Dispose 供测试**  
   ```csharp
   public static void ResetForTest() => I = new AudioConfig();
   ```

------------------------------------------------  
【四不做】  
1. ❌ **不放可变运行时状态**  
   ‑ 例如玩家血量、房间 ID、网络连接状态。  

2. ❌ **不继承 MonoBehaviour**  
   ‑ 避免 `DontDestroyOnLoad` 永久驻留；如需 Mono 生命周期，用普通脚本 + 服务定位器。  

3. ❌ **不在 Awake/Start 里写可变逻辑**  
   ‑ 防止场景反复加载后状态残留。  

4. ❌ **不用 Resources/静态字段缓存 Unity 对象**  
   ‑ 会导致 Addressable 卸载失败；用 `WeakReference` 或对象池。  

------------------------------------------------  
一句话总结  
单例只当“只读工具箱”和“全局常量仓库”；  
凡是**会变的、需要随场景销毁的、需要 Mock 测试的**，交给 **依赖注入 / 服务定位器 / 普通实例**，别让单例背锅。
