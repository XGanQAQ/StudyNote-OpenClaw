# 服务定位器（Service Locator）

## 概念

全局"查号台"：按接口注册服务，按接口取出实现，完全**不 new、不 Find、不耦合**。

## 最小实现

```csharp
public static class Locator
{
    private static readonly Dictionary<Type, object> _services = new();

    public static void Register<T>(T service) => _services[typeof(T)] = service;
    public static T Resolve<T>() => (T)_services[typeof(T)];
    public static void Clear() => _services.Clear();
}
```

## 使用步骤

1. **定义接口**：`public interface IAudioService { void PlaySfx(string name); }`
2. **启动注册**：`Locator.Register<IAudioService>(new FMODAudioService());`
3. **任意使用**：`Locator.Resolve<IAudioService>().PlaySfx("gunshot");`

## 优缺点

| 维度 | 评价 |
|------|------|
| 解耦 | ✅ 低耦合 |
| 可替换 | ✅ 替换实现只需改注册 |
| 生命周期 | ✅ 显式注册/注销 |
| 测试 | ⚠️ 较难（全局静态） |

## Service Locator vs DI

| 模式 | 依赖获取方式 | 耦合度 | 可测试性 |
|------|-------------|--------|---------|
| 依赖注入（DI） | 外部注入 | 低 | 好 |
| 服务定位器 | 内部查找 | 中 | 差 |

## 建议

- 小项目/原型 → 服务定位器（方便灵活）
- 中大型项目 → 依赖注入（结构清晰）
- 游戏项目可**混合使用**：系统级用 DI，UI/控制器用 Locator
