下面列出 **在 4 人合作第三人称射击联机项目** 中最常见、**真正无状态 / 只读** 的服务清单，并给出 **单例实现范例**。只要它们 **不缓存可变数据**、**运行时不写入磁盘**，就可以放心用单例，不会成为隐藏的“全局状态炸弹”。

| 服务             | 作用                   | 是否可变      | 单例示例代码                                                                                         |
| -------------- | -------------------- | --------- | ---------------------------------------------------------------------------------------------- |
| **游戏配置表**      | 武器数值、关卡常量、掉落表        | 只读        | 见下方 `GameConfig`                                                                               |
| **全局常量**       | 层号、Tag、Animator 参数哈希 | 只读        | `public static class Tags { public const string Enemy = "Enemy"; }`                            |
| **本地化表**       | 多语言文本                | 只读        | `Localization.I.Get("ui_restart")`                                                             |
| **版本号 / 构建信息** | 客户端版本、热更地址           | 只读        | `BuildInfo.Version`                                                                            |
| **随机种子生成器**    | 提供确定性的全局随机           | 无状态（只读接口） | `public static class Rand { public static int Seed {get;} = 12345; }`                          |
| **日志封装**       | Debug.Log 统一格式       | 无状态（只写日志） | `public static class Log { public static void I(object msg) => Debug.Log(msg); }`              |
| **时间工具**       | 服务器时间、帧号转换           | 只读        | `public static class TimeUtil { public static long ServerNowMs => NetworkClient.ServerTime; }` |
| **数学公式库**      | 伤害计算、衰减公式            | 纯函数，无状态   | `public static class Formulas { public static float Falloff(float d,float r) => d*(1-r); }`    |

------------------------------------------------
最小单例示例：只读配置表

```csharp
// GameConfig.cs
public sealed class GameConfig
{
    // 1. 单例入口
    public static GameConfig I { get; } = new GameConfig();

    // 2. 私有构造，禁止外部 new
    private GameConfig() { }

    // 3. 运行时只读字段
    public readonly Dictionary<int, WeaponData> Weapons = LoadWeaponTable();

    private static Dictionary<int, WeaponData> LoadWeaponTable()
    {
        // 反序列化 JSON / ScriptableObject，完成后不再写入
        return JsonConvert.DeserializeObject<Dictionary<int, WeaponData>>(Resources.Load<TextAsset>("WeaponTable").text);
    }
}

// 使用
float damage = GameConfig.I.Weapons[weaponId].BaseDamage;
```

------------------------------------------------
一句话总结  
只要数据 **启动时一次性加载、运行期绝不修改**，就可以做成 **单例** 或 **静态类**，既省性能又降低复杂度；其余任何带 **可变缓存、网络状态、玩家数据** 的服务请改用 **依赖注入 / 服务定位器 / 普通实例**。
