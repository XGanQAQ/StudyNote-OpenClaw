# 依赖注入（Dependency Injection）

## 概念

依赖注入 = 把对象需要的东西**从外部递进来**，而不是让它自己去找。

## 生活比喻：点奶茶

- **传统做法**：自己冲进厨房拿杯子、舀珍珠、倒茶（类里 new 或 Find）
- **DI**：前台把配好的奶茶递给你（你只关心喝）

## 代码对比

### 传统写法（耦合噩梦）
```csharp
class Weapon
{
    private IAmmoPool ammo = AmmoPoolSingleton.I;
    private IAudioService audio = FindObjectOfType<AudioManager>();
    private INetSync net = NetClient.I;
}
```

### DI 写法（零耦合）
```csharp
class Weapon
{
    private readonly IAmmoPool ammo;
    private readonly IAudioService audio;
    public Weapon(IAmmoPool ammo, IAudioService audio) { ... }
}
```

## Unity 落地方式

| 方式 | 一句话 | 包 |
|------|--------|-----|
| 手动注入 | 自己 new 时传递依赖 | 无需 |
| Zenject | 最成熟的 Unity DI 框架 | GitHub |
| Microsoft DI | .NET 官方轻量容器 | NuGet |

## 好处

1. **解耦**：只依赖接口
2. **可测试**：注入 Mock 实现
3. **可配置**：同一套代码不同平台注入不同实现
4. **生命周期集中管理**
