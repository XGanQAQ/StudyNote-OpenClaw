把依赖注入（DI）想象成 **“把对象需要的东西从外部递进来，而不是让它自己去找”**。  
在游戏项目里，它能一次性干掉“到处 `FindObjectOfType`、`GetComponent`、`Singleton.X`”带来的耦合和测试噩梦。

下面用 **Unity 场景里的真实例子** + **生活比喻** + **三步代码** 让你 5 分钟看懂。

------------------------------------------------
1. 生活比喻：点奶茶  
• 传统做法：你自己冲进厨房拿杯子、舀珍珠、倒茶 → 类里 `new` 或 `Find`。  
• 依赖注入：前台把**配好的奶茶**递给你 → 你只关心“喝”，不管杯子是谁洗的、珍珠是谁煮的。

------------------------------------------------
2. 游戏例子：武器开火需要三样东西  
- 弹药池 `IAmmoPool`  
- 音效系统 `IAudioService`  
- 网络同步 `INetSync`  

**传统写法**（耦合噩梦）：
```csharp
class Weapon
{
    private IAmmoPool   ammo   = AmmoPoolSingleton.I;   // 强耦合
    private IAudioService audio = FindObjectOfType<AudioManager>();
    private INetSync    net    = NetClient.I;
}
```

**依赖注入写法**（零耦合）：
```csharp
class Weapon
{
    private readonly IAmmoPool ammo;
    private readonly IAudioService audio;
    private readonly INetSync net;

    // 构造函数一次性“收外卖”
    public Weapon(IAmmoPool ammo, IAudioService audio, INetSync net)
    {
        this.ammo   = ammo;
        this.audio  = audio;
        this.net    = net;
    }
}
```
武器类 **完全不关心**这些服务是谁、在哪创建，只管用接口调用。

------------------------------------------------
3. 在 Unity 里落地的 3 种常见方式

| 方式 | 一句话 | NuGet/包 | 代码片段 |
|---|---|---|---|
| **手动构造函数注入** | 自己 new 的时候把依赖塞进去 | 无需包 | `var w = new Weapon(pool, audio, net);` |
| **Zenject/Extenject** | 最成熟的 Unity DI 框架 | GitHub/Zenject | `Container.Bind<IAmmoPool>().To<AmmoPool>().AsSingle();` |
| **Microsoft.Extensions.DependencyInjection** | .NET 官方轻量容器 | 通过 NuGet for Unity | `services.AddSingleton<IAmmoPool, AmmoPool>();` |

示例：Zenject 的最小配置  
```csharp
public class GameInstaller : MonoInstaller
{
    public override void InstallBindings()
    {
        Container.Bind<IAmmoPool>().To<AmmoPool>().AsSingle();
        Container.Bind<IAudioService>().To<AudioManager>().AsSingle();
        Container.Bind<INetSync>().To<NetClient>().AsSingle();
        Container.Bind<Weapon>().AsTransient();   // 每次需要就 new 一个
    }
}
```
然后在任意 MonoBehaviour 里：
```csharp
[Inject] Weapon weapon;   // Zenject 自动把三个依赖“送进来”
```

------------------------------------------------
4. DI 带来的 4 个立竿见影的好处

5. **解耦**：Weapon 不再依赖具体类，只依赖接口。  
6. **测试**：单元测试时可注入 `MockAmmoPool`，无需跑完整场景。  
7. **可配置**：同一套武器代码，PC 版注入 `SteamAudioService`，手机版注入 `FMODAudioService`。  
8. **生命周期集中管理**：单例、瞬时、作用域（per-room）都能一句话配置。

------------------------------------------------
一句话总结  
依赖注入 = **“把依赖从内部 new 变成外部传进来”**，  
配合 **接口 + 容器（Zenject 等）**，  
让游戏代码从“到处找组件”变成“只关心业务逻辑”。
