# 代码的坏味道（Code Smells）

代码坏味道是代码在设计层面上透露出潜在问题的迹象。

## 常见类型

| 类型 | 描述 |
|------|------|
| 重复代码 | 违反 DRY 原则 |
| 过长方法 | 函数过⻓，职责过多 |
| 过大的类 | 承担了太多责任 |
| 神对象（God Object） | 类知道太多、做太多 |
| 发散式变化 | 类因不同原因频繁修改 |
| 散弹式修改 | 一个修改需改动多处 |
| 过度耦合 | 模块间依赖太多 |
| 基本类型偏执 | 过多使用基本类型而非封装对象 |
| 数据泥团 | 一组数据常一起出现，应封装 |
| 消息链 | a.b().c().d() 式调用 |
| 过度注释 | 用注释解释糟糕的代码 |

## 解决方式

- 使用设计模式（工厂、策略等）
- 应用 SOLID 原则
- 提取方法/类
- 减少重复，增强封装
- 使用合适的数据结构
