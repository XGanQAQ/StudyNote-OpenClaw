# 事件总线（EventBus）

## 何时使用

### ✅ 建议用
- **一对多/多对多**通讯（武器发射 → UI 播特效、震屏、音效等）
- **跨层级/跨场景/跨 DLL**
- **生命周期不一致**（订阅方动态创建/销毁）
- **需要热插拔**

### ❌ 不建议用
- **单一对象内部**（如玩家 HP 变化更新自己血条）
- **高实时、高带宽**（每帧上千次更新）
- **需要同步返回值**

## 基础实现

```csharp
public static class EventBus
{
    private static readonly Dictionary<Type, Delegate> _handlers = new();

    public static void Subscribe<T>(Action<T> callback) { ... }
    public static void Unsubscribe<T>(Action<T> callback) { ... }
    public static void Raise<T>(T evt) { ... }
    public static void Clear() => _handlers.Clear();
}
```

## 使用示例

```csharp
// 定义事件（struct，零GC）
public readonly struct PlayerDied { public int KillerId; public int VictimId; }

// 订阅
EventBus.Subscribe<PlayerDied>(OnPlayerDied);
EventBus.Unsubscribe<PlayerDied>(OnPlayerDied);

// 触发
EventBus.Raise(new PlayerDied { KillerId = 1, VictimId = 2 });
```

## 6 条铁律

1. 订阅者在 OnDestroy/Dispose 中必须 Unsubscribe
2. 网络线程来的事件必须回到主线程 Raise
3. 高频事件用 struct，避免 GC
4. 不可依赖事件接收顺序
5. 避免循环触发
6. 加调试面板查看订阅者

## 委托 vs 事件 vs 事件总线

| 类型 | 作用域 | 适用场景 |
|------|--------|----------|
| 委托 Delegate | 字段级 | 回调、排序 |
| 事件 event | 类级 | 按钮点击、属性变更 |
| 事件总线 | 进程级 | 跨模块解耦、跨场景通讯 |

> 委托是列表，事件是保险箱，总线是邮局。
