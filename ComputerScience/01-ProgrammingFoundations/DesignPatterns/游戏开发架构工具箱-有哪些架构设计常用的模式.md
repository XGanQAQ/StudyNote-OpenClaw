下面给出一份 **“大型 Unity 联机射击项目”** 可直接落地的 **架构工具箱**：  
把“事件总线”放在中间，左右各列出 **同级别的 8 个常用基础构建 + 设计模式**，并给出一句话定位、最小示例与适用时机，方便你按场景拼装。

------------------------------------------------  
1. 事件总线（EventBus）【已给实现】  
一句话：全局“邮局”，发布-订阅解耦。  
示例：`EventBus.Raise(new PlayerDied(id));`

------------------------------------------------  
2. 事件队列（Event Queue / 消息队列）  
一句话：把即时调用变成“排队 + 延迟/异步”处理，避免帧率抖动。  
示例：  
```csharp
public class AudioQueue
{
    readonly Queue<PlaySfxCmd> _q = new();
    public void Enqueue(PlaySfxCmd c) => _q.Enqueue(c);
    public void Update() { while (_q.Count>0) Play(_q.Dequeue()); }
}
```
适用：音频、网络包、日志等**可延迟**任务。

------------------------------------------------  
3. 观察者模式（Observer）  
一句话：一对一/一对多，但生命周期同生同灭，不需全局总线。  
示例：  
```csharp
public class Health { public event Action<int> OnChanged; }
```
适用：UI 血条与玩家本体强绑定场景。

------------------------------------------------  
4. 命令模式（Command）  
一句话：把“动作”封装成对象，可撤销、可序列化到网络。  
示例：  
```csharp
public interface ICmd { void Execute(); void Undo(); }
public struct FireCmd : ICmd { public void Execute() => Fire(); ... }
```
适用：帧同步回滚、操作撤销。

------------------------------------------------  
5. 状态机（FSM / 分层状态机 / Behaviour Tree）  
一句话：角色、AI、UI 的“当前状态”管理。  
示例：  
```csharp
StateMachine<Player> fsm = new();
fsm.Add("Run", new RunState());
```
适用：玩家移动、敌人 AI、比赛阶段。

------------------------------------------------  
6. 对象池（Object Pool）  
一句话：复用 GameObject，避免 Instantiate/Destroy 的 GC 与卡顿。  
示例：  
```csharp
public class BulletPool : MonoBehaviour
{
    Queue<Bullet> _pool = new();
    public Bullet Get() => _pool.Count>0 ? _pool.Dequeue() : Instantiate(prefab);
    public void Return(Bullet b) { b.Reset(); _pool.Enqueue(b); }
}
```
适用：子弹、特效、音效源。

------------------------------------------------  
7. 服务定位器（Service Locator）  
一句话：全局“电话簿”，按接口拿服务，比单例更解耦。  
示例：  
```csharp
public static class Locator
{
    public static IAudio Audio { get; set; } = new NullAudio();
}
```
适用：跨平台音频、网络后端切换。

------------------------------------------------  
8. 单例模式（Singleton）  
一句话：全局唯一实例，慎用，仅用于**无状态或只读**服务。  
示例：  
```csharp
public class GameConfig
{
    public static GameConfig I { get; } = new();
    private GameConfig() { }        // 私有构造
}
```
适用：配置表、版本号。

------------------------------------------------  
9. 工厂模式（Factory / 抽象工厂）  
一句话：把“创建”逻辑集中，支持运行时换皮肤、换平台资源。  
示例：  
```csharp
public interface IWeaponFactory { IWeapon Create(int id); }
public class WeaponFactoryPC : IWeaponFactory { ... }
```

------------------------------------------------  
10. CQRS（命令查询职责分离）  
一句话：写（Command）与读（Query）完全分离，网络层高频读缓存。  
示例：  
```csharp
// Command
public record FireCmd(int OwnerId, Vector3 Dir);
// Query
public record GetPlayerPosQuery(int PlayerId) : IRequest<Vector3>;
```

------------------------------------------------  
如何拼装  
• **逻辑层内** → 观察者 / 状态机 / 命令  
• **跨模块** → 事件总线 / 事件队列  
• **资源管理** → 对象池 / 工厂  
• **全局服务** → 单例 or 服务定位器  
• **网络帧同步** → 命令 + 事件队列 + CQRS

把这些“积木”按需组合，就能搭出可维护、可扩展的大型游戏架构。
