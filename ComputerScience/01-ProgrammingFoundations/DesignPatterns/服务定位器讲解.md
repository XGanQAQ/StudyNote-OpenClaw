把“服务定位器”想成 **游戏里的“114 查号台”**：  
谁需要服务，打一通电话（调用接口方法）就能拿到，而不用自己满世界找。  
下面用 3 分钟、3 句话、3 段代码、3 个坑，让你彻底明白。

------------------------------------------------
1️⃣ 一句话定义  
服务定位器 = **全局注册表** + **接口/基类**；  
注册时把“实现”塞进去，使用时按“接口”取出来，**完全不 new、不 Find、不耦合**。

------------------------------------------------
2️⃣ 最小可运行代码（Unity 纯 C#，零依赖）

```csharp
using System;
using System.Collections.Generic;

public static class Locator
{
    private static readonly Dictionary<Type, object> _services = new();

    // 注册
    public static void Register<T>(T service) => _services[typeof(T)] = service;

    // 解析
    public static T Resolve<T>() => (T)_services[typeof(T)];

    // 注销
    public static void Unregister<T>() => _services.Remove(typeof(T));

    // 一键清空（测试用）
    public static void Clear() => _services.Clear();
}
```

------------------------------------------------
3️⃣ 在 Unity 项目中 3 步落地

① 定义接口  
```csharp
public interface IAudioService { void PlaySfx(string name); }
```

② 启动时注册  
```csharp
// 在初始化脚本里
Locator.Register<IAudioService>(new FMODAudioService());
// 或 Addressable 异步加载后注册
Addressables.LoadAssetAsync<GameObject>("AudioManager")
            .Completed += op => Locator.Register<IAudioService>(op.Result.GetComponent<IAudioService>());
```

③ 任意脚本使用  
```csharp
public class Weapon : MonoBehaviour
{
    private IAudioService _audio;

    void Awake()
    {
        _audio = Locator.Resolve<IAudioService>(); // 不需要 FindObjectOfType
    }

    void Fire() => _audio.PlaySfx("gunshot");
}
```

------------------------------------------------
4️⃣ 3 个常见坑 & 规避

| 坑 | 症状 | 规避 |
|---|---|---|
| **忘记注册** | 运行时 `KeyNotFoundException` | 启动流程里统一注册；未注册时返回“空对象”实现 |
| **生命周期失控** | 场景卸载后仍持有引用 | 在 `OnDestroy`/`Shutdown` 里 `Locator.Unregister<T>()` |
| **线程安全** | 多线程注册/解析竞态 | 加 `lock` 或用 `ConcurrentDictionary` |

------------------------------------------------
5️⃣ 一句话对比

| 方式 | 耦合 | 可替换 | 生命周期 |
|---|---|---|---|
| `FindObjectOfType<AudioManager>()` | 高 | 难 | 不可控 |
| 单例 `AudioManager.I` | 中 | 难 | 永久 |
| 服务定位器 | 低 | 易 | 显式注册/注销 |

------------------------------------------------
总结  
服务定位器 = **“按接口查服务”的电话簿**。  
注册一次，到处用接口，**不 new、不耦合、可随时替换实现**，是 Unity 项目里最简单、最实用的解耦工具之一。
