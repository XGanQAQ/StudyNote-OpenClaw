# 🧠 Windows 任务计划程序（Task Scheduler）笔记

---

## 📌 一、它是干嘛的？

本质就是一个 **“定时 + 条件触发的自动执行器”**

你可以理解成：

> 👉 Windows 内置的“轻量级定时任务系统”（类似 Linux 的 cron）

---

## 📌 二、核心组成（重点！）

一个任务 = **触发器 + 操作 + 条件 + 设置**

### 1️⃣ 触发器（Triggers）👉 “什么时候执行”

常见类型：

- 🕒 按时间
    
    - 每天 / 每周 / 每月
        
- 🚀 开机时
    
- 🔐 用户登录时
    
- ⚡ 系统空闲时
    
- 📊 事件触发（高级）
    

👉 示例：

- 每天 10 点执行
    
- 登录电脑就执行
    
- 程序崩溃后触发（事件日志）
    

---

### 2️⃣ 操作（Actions）👉 “执行什么”

最常用：

- 启动程序（.exe / .bat / .ps1）
    

也支持：

- 发邮件（已基本废弃）
    
- 显示消息（已废弃）
    

👉 示例：

- 启动 Unity 自动打包脚本
    
- 执行 Python 数据分析
    
- 跑一个 .bat 自动部署
    

---

### 3️⃣ 条件（Conditions）👉 “在什么情况下才执行”

可选限制：

- 仅在 **接通电源** 时运行
    
- 仅在 **网络可用** 时运行
    
- 仅在 **空闲状态** 执行
    

---

### 4️⃣ 设置（Settings）👉 “执行策略”

关键选项：

- ❗ 失败后重试
    
- ⏱ 超时自动停止
    
- 🔁 任务已在运行时：
    
    - 忽略新实例
        
    - 并行运行
        
    - 停止旧任务
        

---

## 📌 三、常见使用场景（结合你开发方向）

### 🎮 游戏开发 / 程序员常用

- 自动构建（Build）

- 自动运行测试脚本
    

👉 示例：

```
每天凌晨 3 点：
→ 拉代码

```

### 💻 日常自动化

- 开机自动启动工具

---

## 📌 四、创建任务（实操流程）

### 推荐用：👉 **“创建任务”（不是“创建基本任务”）**

---

### 🛠 示例：定时执行脚本

1. 打开任务计划程序
    
2. 点击 **创建任务**
    

---

#### 【常规】

- 名称：`AutoBuild`
    
- ✔ 勾选：
    
    - 使用最高权限运行（重要）
        

---

#### 【触发器】

- 新建 → 每天 → 03:00
    

---

#### 【操作】

- 程序/脚本：
    
    ```
    powershell
    ```
    
- 参数：
    
    ```powershell
    -File "D:\scripts\build.ps1"
    ```
    

---

#### 【条件】

- 根据需要选择（比如必须联网）
    

---

#### 【设置】

- ✔ 失败后重试
    
- ✔ 允许运行超过计划时间
    

---

## 📌 五、命令行方式（更适合你）

---

### 👉 查看任务

```powershell
Get-ScheduledTask
```

---

### 👉 创建任务（CLI）

```cmd
schtasks /create /tn "MyTask" /tr "notepad.exe" /sc daily /st 10:00
```

---

### 👉 删除任务

```cmd
schtasks /delete /tn "MyTask" /f
```

---

### 👉 手动执行

```cmd
schtasks /run /tn "MyTask"
```

---

## 📌 六、常见坑（非常重要）

### ⚠️ 1. 路径问题

- 相对路径 ❌
    
- 一定用 **绝对路径**
    

---

### ⚠️ 2. 权限问题

- 勾选：
    
    > ✔ 使用最高权限运行
    

---

### ⚠️ 3. PowerShell脚本不执行

需要加：

```powershell
-ExecutionPolicy Bypass
```

---

### ⚠️ 4. 程序一闪而过

👉 建议：

- 输出日志到文件
    
- 或用 `.bat` 包装
    

---

### ⚠️ 5. 任务“运行了但没效果”

👉 常见原因：

- 工作目录错误
    
- 环境变量不一致
    
- 用户权限问题
