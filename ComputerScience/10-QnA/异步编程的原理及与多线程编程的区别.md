# 异步编程的原理与多线程的区别

## 异步编程原理

异步编程是一种**非阻塞的编程模型**，允许程序在等待任务完成时继续执行其他代码。

### 核心机制

1. **事件循环（Event Loop）**：持续监听事件，任务完成后将结果推送到回调队列
2. **回调机制**：注册回调函数，任务完成后触发
3. **任务调度器**：负责分发任务的执行
4. **状态机**：跟踪任务状态（待执行、执行中、已完成）
5. **Promise/Future**：表示未来结果的占位符对象

## 异步 vs 多线程

| 特性 | 异步编程 | 多线程编程 |
|------|---------|-----------|
| 核心思想 | 非阻塞，任务切换 | 并发执行多个线程 |
| 线程使用 | 少量线程，甚至单线程 | 每个线程独立分配资源 |
| 实现方式 | 事件循环、回调、任务队列 | 操作系统线程调度 |
| 性能特点 | 适合 I/O 密集型 | 适合 CPU 密集型 |
| 资源消耗 | 较少 | 每个线程独立栈 |
| 关注点 | 任务流控制 | 线程同步与竞争 |

## 适用场景

### 异步编程
- I/O 密集型任务（网络请求、文件读写、数据库访问）
- 延迟加载、后台数据同步
- 单线程高并发（如 Node.js）

### 多线程编程
- CPU 密集型任务（图像处理、科学计算）
- 需要真正并行性（渲染与逻辑分离）
- 多核 CPU 提升执行效率

## 混合使用

现代应用常将两者结合使用：
- **Web 服务**：异步处理 I/O，多线程处理业务逻辑
- **游戏开发**：异步加载资源，多线程处理物理计算
- **高性能服务器**：异步框架 + 线程池处理密集计算

**总结：** 异步编程适合大量 I/O 请求的非阻塞处理；多线程注重利用多核 CPU 实现并行计算。
