# 操作系统

## 什么是操作系统
操作系统是管理硬件资源、为应用程序提供服务的软件层。

### 操作系统的三块核心
| 方面 | 说明 |
|------|------|
| **虚拟化** | CPU→进程、内存→地址空间、存储→文件系统 |
| **并发** | 多线程/多进程、同步互斥、死锁 |
| **持久化** | 文件系统、设备驱动、存储管理 |

## 应用视角的操作系统（南大 OS 课程视角）
- 程序就是**状态机**：内存+寄存器 → 指令→Syscall 构成状态转移
- 系统调用是程序与操作系统的唯一接口
- `strace` 可跟踪程序的系统调用行为

### 常用调试命令
```bash
strace ./program     # 跟踪系统调用
objdump -d program   # 反汇编
gdb program          # 调试器
file program         # 查看文件类型
```

## 虚拟化

### 进程管理
- `fork()`：创建子进程（复制状态机）
- `execve()`：替换当前进程（重置状态机）
- `waitpid()`：等待子进程结束
- 进程地址空间：mmap 映射内存、栈/堆管理

### 可执行文件与链接
- ELF 格式：可执行与链接格式
- 静态链接 vs 动态链接
- libc：标准 C 库的实现

## 并发（多线程）

### 线程模型
- 共享内存、独立堆栈
- 原子性（Atomicity）：需要锁来保证
- 顺序（Ordering）：内存模型决定可见性

### 同步机制
| 方式 | 说明 |
|------|------|
| 互斥锁 (Mutex) | 保护临界区 |
| 条件变量 (CV) | 生产者-消费者模型 |
| 信号量 (Semaphore) | 资源计数同步 |
| 读写锁 | 读共享写独占 |

### 常见并发 Bug
- 竞态条件 (Race Condition)
- 死锁 (Deadlock)
- 活锁 (Livelock)
- 优先级反转

## 持久化

### 文件系统
- 目录树管理 API：open, read, write, close, opendir
- VFS（虚拟文件系统层）
- ext4 / XFS / Btrfs 等具体实现
- 文件描述符、inode、软/硬链接

### 设备驱动
- 设备抽象为文件（UNIX 哲学）
- 字符设备 vs 块设备
- 中断处理、DMA
