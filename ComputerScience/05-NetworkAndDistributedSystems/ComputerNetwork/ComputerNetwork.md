# 计算机网络

## 分层模型 (OSI / TCP/IP)

### OSI 七层模型
| 层级 | 名称 | 协议/设备 | 说明 |
|------|------|----------|------|
| 7 | 应用层 | HTTP/FTP/SMTP | 用户接口 |
| 6 | 表示层 | SSL/TLS | 数据格式化、加密 |
| 5 | 会话层 | NetBIOS | 会话管理 |
| 4 | 传输层 | TCP/UDP | 端到端通信 |
| 3 | 网络层 | IP/ICMP | 路由、寻址 |
| 2 | 数据链路层 | Ethernet/PPP | 帧传输 |
| 1 | 物理层 | 网线/信号 | 比特流传输 |

### TCP/IP 四层模型（实际应用）
| 层级 | 协议 |
|------|------|
| 应用层 | HTTP, HTTPS, DNS, FTP, SMTP |
| 传输层 | TCP, UDP |
| 网络层 | IP, ICMP, ARP |
| 网络接口层 | Ethernet, WiFi |

## 核心协议

### TCP
- 面向连接、可靠传输
- 三次握手、四次挥手
- 流量控制：滑动窗口
- 拥塞控制：慢启动、拥塞避免、快重传、快恢复

### UDP
- 无连接、不可靠
- 低延迟，适合实时应用（视频、游戏）

### HTTP/HTTPS
- HTTP 无状态、请求-响应模型
- HTTPS = HTTP + TLS/SSL
- 状态码：2xx成功, 3xx重定向, 4xx客户端错误, 5xx服务端错误

### DNS
- 域名→IP 地址解析
- 层次结构：根→顶级域→二级域→子域

## 数据链路层
- PPP 协议：点对点链路控制
- 数据链路层三大功能：**封装成帧、透明传输、差错检测**
- MAC 地址：物理地址
- CSMA/CD：以太网冲突检测

## 计算机网络编程
- Socket API
- 阻塞/非阻塞 I/O
- select/poll/epoll（I/O 多路复用）
- 网络编程常见模型：单线程、多线程、Reactor、Proactor
