# Utils 模块职责规范（精简版）

## 1. 模块定位

**Utils 是框架的基础工具层，用于为 Core 及上层模块提供通用、可复用的辅助能力。**

Utils 面向代码使用，不面向开发者操作。

---

## 2. 职责范围

Utils 负责提供：

- 通用工具与辅助功能  
    （时间、数学、随机、路径、数据转换等）
    
- 轻量级、无业务语义的功能封装
    
- 可被多个模块复用的基础能力
    

---

## 3. 设计要求

Utils 中的代码必须满足：

- 与具体业务无关
    
- API 简单、稳定
    
- 可独立使用
    
- 不管理系统流程或生命周期
    

---

## 4. 非职责（禁止事项）

Utils **不得**：

- 包含业务逻辑或玩法规则
    
- 管理初始化、Update、状态切换等流程
    
- 依赖 Core 及其子模块
    
- 使用 `UnityEditor` 相关 API
    
- 提供编辑器工具或 UI
    

---

## 5. 依赖规则

```text
Utils  →  无
Core   →  Utils
Tools  →  Utils / Core
```

Utils 只能被依赖，不能反向依赖。

---

## 6. 命名与内容约定

- 命名体现功能，不体现业务
    
    - ✅ `UniTimer`
        
    - ❌ `BattleTimer`
        
- 常见内容：
    
    - Time / Math / IO / Random / Pool / Extensions
        

---

## 7. 判定标准

如果一个类满足以下条件，则应放入 Utils：

- 对玩家无直接意义
    
- 不依赖具体系统
    
- 在多个模块中都有使用价值
    
- 即使 Core 精简后仍然成立
    

---

## 8. 总结

> **Utils 是稳定的基础工具箱，  
> 为系统服务，但不成为系统本身。**
