依赖管理的核心问题

> 如何在不破坏模块独立性的前提下，让不同系统之间互相访问？

所以我们需要一个**依赖管理机制**来：
- 让系统之间**低耦合**
- 让依赖**明确可控**
- 方便调试、扩展、测试

## 分层混合方案
“单例 + 服务定位器 + 局部依赖注入（使用构造函数注入（轻量实现，不依赖外部框架））” 混合使用。

- 底层系统：单例 (MonoSingleton)
	- 底层系统
	- 生命周期与游戏同在，即游戏一开始就存在，且
	- 便于直接通过 `.Instance` 调用；
	- 但不要在逻辑层直接互相依赖（通过 Locator 调用更好）**保持独立！！！**
- 中间层：服务定位器 (ServiceLocator)
	- 只承担注册与分发
	- UI获得常见中加载控制器
	- 单例与单例之间的引用获取
- 逻辑层 / 模块层：依赖注入 (DI)
	- 手动DI构造函数


我大概预设了这样的使用场景，程序一开始，会加载一些常驻单例，在加载到Gameplay场景的时候玩家控制器会注册到服务定位器中，此时UI可以从服务定位器获得此控制器，进行相关数据显示

## 🧭 三种主要方案对比

|特性|单例（Singleton / MonoSingleton）|服务定位器（ServiceLocator）|依赖注入（Dependency Injection）|
|---|---|---|---|
|**访问方式**|直接访问 `AudioManager.Instance`|通过容器访问 `ServiceLocator.Get<AudioManager>()`|外部传入构造或字段 `[Inject] AudioManager audio;`|
|**控制反转 (IoC)**|❌ 无反转（对象主动控制依赖）|⚙️ 部分反转（依赖隐藏）|✅ 完全反转（外部装配依赖）|
|**耦合度**|高|中|低|
|**易用性**|简单|中等|较复杂|
|**可测试性**|很差|一般|非常好|
|**适合规模**|小项目 / 原型|中项目 / 学生团队|中大型项目 / 长线维护|
|**常见问题**|全局状态污染|隐式依赖|配置麻烦、需要框架支持|
|**典型代表**|`GameManager.Instance`|自写轻量 Locator|Zenject / VContainer / Extenject|

## 🎯 对比总结表

| 特性                 | 普通单例     | MonoSingleton | ServiceLocator |
| ------------------ | -------- | ------------- | -------------- |
| 是否继承 MonoBehaviour | ❌        | ✅             | ❌（可选）          |
| 是否依赖场景             | ❌        | ✅             | 可选             |
| 生命周期               | 全局       | 场景/全局对象       | 可定制（模块化）       |
| 可否持有 Unity 组件      | ❌        | ✅             | ✅（如果注册Mono类型）  |
| 是否支持解耦             | ❌        | ⚠️ 一般         | ✅ 高度解耦         |
| 是否支持多实例（模块化）       | ❌        | ❌             | ✅ 可多层级         |
| 是否可热替换             | ❌        | ⚠️ 难          | ✅ 动态注册注销       |
| 实现复杂度              | ⭐        | ⭐⭐            | ⭐⭐⭐            |
| 推荐用途               | 数据/逻辑类管理 | Unity系统类管理    | 跨模块服务管理        |

## 💡 实战建议（Unity项目中）

|使用目标|推荐方案|
|---|---|
|全局配置、数据缓存、工具类|✅ **普通单例**|
|需要 Unity 生命周期 / 场景存在感（如声音、输入、UI）|✅ **MonoSingleton**|
|管理多种类型的系统服务（模块级依赖解耦）|✅ **ServiceLocator**|
|UI 控制器或子系统的统一注册和访问|✅ `ViwerControllerLocator`（模块化ServiceLocator）|

## ⚙️ 举例对比

| 需求                  | 适用模式           | 举例                                                        |
| ------------------- | -------------- | --------------------------------------------------------- |
| 游戏配置常驻全局            | 普通单例           | `GameConfig.Instance.MaxPlayerCount`                      |
| 音效管理（需要AudioSource） | MonoSingleton  | `AudioManager.Instance.Play("explosion")`                 |
| 游戏系统模块间解耦调用         | ServiceLocator | `Locator.Resolve<IPlayerService>().Attack()`              |
| UI 控制器注册与访问         | ServiceLocator | `UIControllerLocator.Instance.TryGetWait<MainMenuCtrl>()` |
