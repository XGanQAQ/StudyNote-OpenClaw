`OnValidate()` 是 Unity 在 **Editor 模式** 下才会调用的一个 **生命周期钩子**，**发布版（Build）里不会进这个函数**。  
一句话：**让你在 Inspector 里改字段时立刻“预览”效果或做合法性检查**，而不用手动运行游戏。

--------------------------------------------------
1. 触发时机
--------------------------------------------------
- 第一次在 Inspector 里赋值组件  
- 之后在面板里改任何序列化字段（直接输入、拖引用、勾布尔）  
- 保存场景 / 保存 Project 时也会再走一次  
- 脚本被重新编译后

--------------------------------------------------
2. 典型用途
--------------------------------------------------

| 目的       | 示例代码                                                                                                   |
| -------- | ------------------------------------------------------------------------------------------------------ |
| **即时预览** | 改颜色立刻刷新材质：`myRenderer.material.color = color;`                                                         |
| **范围约束** | 把血量 clamp 到 0~100：`health = Mathf.Clamp(health, 0, 100);`                                              |
| **依赖注入** | 字段为空自动找同级组件：`if (rb == null) rb = GetComponent<Rigidbody>();`                                          |
| **调试开关** | 我在 `TimePauseManager` 里用 `_pauseToggle` 布尔，在 Editor 里打钩就立刻 `SetPaused(true)`，不打钩就 `false`，不用运行就能看暂停效果。 |

--------------------------------------------------
3. 注意事项
--------------------------------------------------
1. **只在 Editor 有效**：  
   `Application.isPlaying == false` 时也能进；若只想运行时生效，记得加判断。  
2. **不要在这里创建/销毁资源**：  
   因为会频繁调用，可能导致泄漏或 Undo 异常；如需生成/销毁，用 `EditorApplication.delayCall += () => { ... };` 延迟执行。  
3. **字段必须是序列化的**：  
   `public` 或 `[SerializeField] private` 才会被 Inspector 识别并触发。  
4. **避免无限递归**：  
   在 `OnValidate` 里改同字段值不会再触发自身，但改别的字段可能再次进入，逻辑要简明。  

--------------------------------------------------
4. 最小可运行示例
--------------------------------------------------
```csharp
using UnityEngine;

public class ValidatorExample : MonoBehaviour
{
    [Range(0, 10)]
    public int size = 1;

    private void OnValidate()
    {
        // 立刻把 size 应用到 transform
        transform.localScale = Vector3.one * size;
    }
}
```
把脚本挂到任意物体，在 Inspector 拖动 `size` 滑块，场景里的立方体会实时变大变小——**游戏没运行就能看到效果**。

--------------------------------------------------
总结
--------------------------------------------------
`OnValidate` 就是 Unity 给开发者提供的“**Inspector 即时回调**”，用来：  
- 做数据合法性检查  
- 立即预览视觉/逻辑效果  
- 自动补全缺失引用  

**正式发布后它不存在**，所以只把“编辑器辅助”逻辑放这里，运行时代码不要依赖它。
